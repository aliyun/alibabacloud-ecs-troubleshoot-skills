| 使用类问题 | 磁盘分区表、分区和文件系统损坏或者不符合预期 | GuestOS.CorruptDiskStorage         | 磁盘通常需分区表/分区与文件系统才能访问。分区表常见 MBR/GPT；文件系统常见 ext4/xfs。损坏或挂载不符预期可导致启动异常、服务或业务异常。参见 [分区表类型](https://access.redhat.com/documentation/zh-cn/red_hat_enterprise_linux/9/html/managing_storage_devices/comparison-of-partition-table-types_disk-partitions)、[文件系统概述](https://access.redhat.com/documentation/zh-cn/red_hat_enterprise_linux/9/html/managing_file_systems/overview-of-available-file-systems_managing-file-systems)、[初始化数据盘](https://help.aliyun.com/zh/ecs/user-guide/overview-27) | 数据无法访问、文件系统损坏、磁盘无法挂载或未正确挂载、数据丢失、文件系统信息丢失等。参见 [块存储 FAQ](https://help.aliyun.com/zh/ecs/user-guide/faq-4)                                               | [disk-fs-damaged.md](disk-fs-damaged.md)                           | dms-instancedefault |
| 使用类问题 | 时钟异常跳变                 | GuestOS.AbnormalClockDrift         | 实例有系统时钟与硬件时钟（RTC，虚拟化模拟）。RTC 初值由镜像 standardizedTimeZone 决定（true→UTC，false→北京时间）。系统有时区与 RTC on localtime；需与 standardizedTimeZone 一致否则易错乱。另有 NTP/chrony、时区固化等。参见 [NTP](https://help.aliyun.com/zh/ecs/user-guide/alibaba-cloud-ntp-server)、[Linux 时间与时区](https://help.aliyun.com/zh/ecs/user-guide/linux-time-and-time-zones)                                                                                                                                                                | 时间同步异常、重启后时间跳变等。                                                                                                                                      | [clock-abnormal.md](clock-abnormal.md)                             | dms-instancedefault |
| 使用类问题 | 实例无法连通网络            | GuestOS.OutsideNetworkAccessFailed | 实例连通网络至少需：网卡 IP、路由、DNS。主/辅助网卡与辅助 IP 均需配置；VPC 常用阿里云 DNS 100.100.2.136/138。参见 [辅助网卡获取 IP](https://help.aliyun.com/zh/ecs/user-guide/configure-a-secondary-eni)、[分配辅助私网 IP](https://help.aliyun.com/zh/ecs/user-guide/assign-secondary-private-ip-addresses)                                                                                                                                                                                                                                 | cloud-init 无法连元数据、DHCP 配置后不通等。                                                                                                                        | [internal-network-failed.md](internal-network-failed.md)           | dms-instancedefault |
| 使用类问题 | 无法通过 VNC 登录实例          | GuestOS.VNCLoginFailed             | VNC 为 ECS 管控提供的图形化登录方式，不依赖实例网络，故 GuestOS 未启动时也可用 VNC 做故障恢复。但 VNC 通路非 100% 稳定，硬件/软件/架构不支持等可导致无法连接；若需登录 OS 执行运维，则 OS 启动、PAM、Shell 等异常也会导致登录失败。参见 [通过 VNC 连接实例](https://help.aliyun.com/zh/ecs/user-guide/log-on-to-an-instance-by-using-vnc)                                                                                                                                                                                                                                                | VNC 连接后黑屏等。参见 [VNC/Workbench 连接问题](https://help.aliyun.com/zh/ecs/support/through-vnc-or-workbench-instance-remote-connection-problems#705d692095lw1) | [vnc-login-failed.md](vnc-login-failed.md)                         | dms-instancedefault |
| 使用类问题 | 无法通过 SSH 远程登录实例        | GuestOS.SSHConnectingFailed        | SSH 提供加密远程连接与命令执行。建联失败可能因 OS 未启动、网络不通、防火墙、sshd 配置、PAM、Shell 或文件异常等。参见 [SSH 服务介绍](https://help.aliyun.com/zh/ecs/ssh-service-introduction)                                                                                                                                                                                                                                                                                                                                                 | SSH 远程登录实例失败、连接报错。参见 [SSH 远程连接问题](https://help.aliyun.com/zh/ecs/support/ssh-remote-connection-problems)                                              | [ssh-login-failed.md](ssh-login-failed.md)                         | dms-instancedefault, dms-remoteconnectdiagnose |


## 宕机或夯机类问题


| 现象域大类    | 现象域  | 唯一标识          | 概念解释                                                                                                                                                                                                                                                                                                                                                                            | 典型现象                                                        | 排查流程                               | 推荐的诊断集合 |
| -------- | ---- | ------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------- | ---------------------------------- | ------- |
| 宕机或夯机类问题 | 系统夯机 | GuestOS.Hang  | 夯机指进程或系统停止响应输入（键盘/鼠标）。原因多样：软件或硬件缺陷、无限循环、资源耗尽（如磁盘忙）、硬件限速、网络慢、配置或兼容问题；根因常为资源耗尽。可为暂时自愈或需人工干预。与宕机不同，宕机为瞬时发生。                                                                                                                                                                                                                                                                        | 如屏幕出现 「watchdog: BUG: soft lockup - CPU#6 stuck for 23s!」等。 | [system-hang.md](system-hang.md)   | dms-instancedefault |
| 宕机或夯机类问题 | 系统宕机 | GuestOS.Crash | 宕机为内核检测到严重错误时的安全措施，需手动或配置自动重启。开启 kdump 时可自动内存转储便于分析。参见 [收集宕机转储](https://help.aliyun.com/zh/ecs/user-guide/collect-kdump-information-after-an-instance-experiences-an-operating-system-failure)、[宕机排查](https://help.aliyun.com/zh/ecs/support/troubleshoot-instance-panic-issues)、[开启 Kdump](https://help.aliyun.com/zh/ecs/how-to-enable-the-kdump-service-for-linux-instances) | 实例自动关机重启等。 | [system-crash.md](system-crash.md) | dms-unexpectedreboot |

| 性能类问题 | 磁盘 IOPS 异常升高               | GuestOS.DiskIOPSTooHigh                  | IOPS 为每秒读写请求数，异常升高可能导致存储瓶颈、系统或应用变慢，可由高并发读写、碎片整理、内核活动等导致。                                       | 业务进程存储性能下降等。                         | [disk-iops-high.md](disk-iops-high.md)                   | dms-instancedefault |
| 性能类问题 | load1m/load5m/load15m 异常升高 | GuestOS.CPULoadTooHigh                   | load 为平均活跃进程数（运行+不可中断），反映系统整体负载。load1/5/15 为过去 1/5/15 分钟平均值。等于逻辑 CPU 数表示刚好满负载，大于则负载较重。         | 进程运行卡顿等。                             | [load-high.md](load-high.md)                             | dms-instancedefault |
| 性能类问题 | 内存使用率高         | GuestOS.OutOfMemory                      | 内存使用率接近上限时 OS 会回收内存；回收后仍无法分配则按 OOM 算法杀高占用进程。                                                   | 进程突然挂掉；dmesg 有 oomkill 记录。           | [memory-oom.md](memory-oom.md)                           | dms-instancedefault |
| 性能类问题 | 磁盘读写性能不符合预期                | GuestOS.UnexpectedlyDiskIOPerformance    | 磁盘读写性能不满足 [实例性能 SLA](https://help.aliyun.com/zh/ecs/user-guide/overview-of-instance-families)。 | 磁盘 IOPS 上不去、读写速率太慢。                  | [disk-perf-unexpected.md](disk-perf-unexpected.md)       | dms-instancedefault |
| 性能类问题 | CPU 使用率异常升高                | GuestOS.CPUUtilizationTooMuch            | CPU 使用率为处理器执行进程/任务的时间比例，异常升高可由密集程序、配置或异常进程导致。                                                  | 进程运行卡顿等。                             | [cpu-high.md](cpu-high.md)                               | dms-instancedefault |
